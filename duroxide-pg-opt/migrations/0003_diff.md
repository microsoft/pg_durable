# Migration 0003: Add Capability Filtering Support

## Summary

This migration adds support for dispatcher capability filtering, allowing dispatchers to filter orchestration items by pinned duroxide version. This enables safe rolling upgrades where different dispatchers can handle different version ranges.

## Schema Changes

### Tables Modified

#### `executions`
| Change | Column | Type | Description |
|--------|--------|------|-------------|
| ADD | `duroxide_version_major` | `INTEGER` | Major version of pinned duroxide version |
| ADD | `duroxide_version_minor` | `INTEGER` | Minor version of pinned duroxide version |
| ADD | `duroxide_version_patch` | `INTEGER` | Patch version of pinned duroxide version |

```sql
-- Delta from 0002:
ALTER TABLE executions ADD COLUMN IF NOT EXISTS duroxide_version_major INTEGER;
ALTER TABLE executions ADD COLUMN IF NOT EXISTS duroxide_version_minor INTEGER;
ALTER TABLE executions ADD COLUMN IF NOT EXISTS duroxide_version_patch INTEGER;
```

## Stored Procedures

### Modified Procedures

#### `fetch_orchestration_item(p_now_ms, p_lock_timeout_ms, p_min_version_packed, p_max_version_packed)`

**Change**: Now accepts 4 parameters (up from 2). New parameters `p_min_version_packed` and `p_max_version_packed` enable version-based filtering before lock acquisition. Old 2-param signature is dropped.

**Diff** (signature):
```diff
- CREATE OR REPLACE FUNCTION %I.fetch_orchestration_item(
-     p_now_ms BIGINT,
-     p_lock_timeout_ms BIGINT
- )
+ CREATE OR REPLACE FUNCTION %I.fetch_orchestration_item(
+     p_now_ms BIGINT,
+     p_lock_timeout_ms BIGINT,
+     p_min_version_packed BIGINT DEFAULT NULL,
+     p_max_version_packed BIGINT DEFAULT NULL
+ )
```

**Diff** (Phase 1 candidate selection):
```diff
  -- Phase 1: Find a candidate instance (no FOR UPDATE yet)
+ IF p_min_version_packed IS NOT NULL THEN
+     -- Version-filtered path: join to instances and executions to check pinned version
+     SELECT q.instance_id INTO v_instance_id
+     FROM %I.orchestrator_queue q
+     LEFT JOIN %I.instances i ON q.instance_id = i.instance_id
+     LEFT JOIN %I.executions e ON i.instance_id = e.instance_id
+       AND i.current_execution_id = e.execution_id
+     WHERE q.visible_at <= TO_TIMESTAMP(p_now_ms / 1000.0)
+       AND NOT EXISTS (
+         SELECT 1 FROM %I.instance_locks il
+         WHERE il.instance_id = q.instance_id AND il.locked_until > p_now_ms
+       )
+       AND (
+         e.duroxide_version_major IS NULL
+         OR (e.duroxide_version_major * 1000000 + e.duroxide_version_minor * 1000 + e.duroxide_version_patch)
+            BETWEEN p_min_version_packed AND p_max_version_packed
+       )
+     ORDER BY q.visible_at, q.id
+     LIMIT 1;
+ ELSE
      -- No filter: original behavior
      SELECT q.instance_id INTO v_instance_id
      FROM %I.orchestrator_queue q
      ...
+ END IF;
```

**Diff** (Phase 3 re-verification — same version filter applied):
```diff
  -- Phase 3: Re-verify with FOR UPDATE
+ IF p_min_version_packed IS NOT NULL THEN
+     SELECT q.instance_id INTO v_instance_id
+     FROM %I.orchestrator_queue q
+     LEFT JOIN %I.instances i ...
+     LEFT JOIN %I.executions e ...
+     WHERE ... AND (version filter)
+     FOR UPDATE OF q SKIP LOCKED;
+ ELSE
      -- original re-verify
+ END IF;
```

**Version packing format**: `major * 1_000_000 + minor * 1_000 + patch` (e.g., `0.1.17` → `1017`)

**NULL version semantics**: Executions with `duroxide_version_major IS NULL` are always compatible (backward compatible with pre-filtering instances).

#### `ack_orchestration_item(p_lock_token, p_now_ms, p_execution_id, p_history_delta, p_worker_items, p_orchestrator_items, p_metadata, p_cancelled_activities)`

**Change**: Now extracts and stores `pinned_duroxide_version` from metadata JSON into execution version columns.

**Diff** (new block after status update):
```diff
+ -- Store pinned duroxide version on execution if provided
+ IF p_metadata ? 'pinned_duroxide_version' AND p_metadata->'pinned_duroxide_version' IS NOT NULL
+    AND p_metadata->>'pinned_duroxide_version' != 'null' THEN
+     UPDATE %I.executions
+     SET duroxide_version_major = (p_metadata->'pinned_duroxide_version'->>'major')::INTEGER,
+         duroxide_version_minor = (p_metadata->'pinned_duroxide_version'->>'minor')::INTEGER,
+         duroxide_version_patch = (p_metadata->'pinned_duroxide_version'->>'patch')::INTEGER
+     WHERE instance_id = v_instance_id AND execution_id = p_execution_id;
+ END IF;
```

## Breaking Changes

None. All changes are additive:
- New columns have no NOT NULL constraint (NULL = always compatible)
- Old 2-param `fetch_orchestration_item` is dropped but new 4-param version has defaults (`NULL`) preserving backward compatibility for SQL callers
- `ack_orchestration_item` signature is unchanged; version handling triggered only when `pinned_duroxide_version` key exists in metadata JSON

## Provider Code Changes Required

The Rust provider must:

1. Pass `pinned_duroxide_version` in the metadata JSON to `ack_orchestration_item`:
```rust
let metadata_json = serde_json::json!({
    "orchestration_name": metadata.orchestration_name,
    "orchestration_version": metadata.orchestration_version,
    "status": metadata.status,
    "output": metadata.output,
    "parent_instance_id": metadata.parent_instance_id,
    "pinned_duroxide_version": metadata.pinned_duroxide_version.as_ref().map(|v| {
        serde_json::json!({
            "major": v.major,
            "minor": v.minor,
            "patch": v.patch,
        })
    }),
});
```

2. Pass version filter parameters to `fetch_orchestration_item`:
```rust
sqlx::query_as(&format!(
    "SELECT * FROM {}.fetch_orchestration_item($1, $2, $3, $4)",
    self.schema_name
))
.bind(now_ms)
.bind(lock_timeout_ms)
.bind(min_packed)  // Option<i64>
.bind(max_packed)  // Option<i64>
```
