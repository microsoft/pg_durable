# Migration 0002: Add Deletion and Pruning Support

## Summary

This migration adds support for instance hierarchy tracking, cascade deletion, and execution pruning. These features enable the `ProviderAdmin` trait methods for instance lifecycle management.

## Schema Changes

### Tables Modified

#### `instances`
| Change | Column | Type | Description |
|--------|--------|------|-------------|
| ADD | `parent_instance_id` | `TEXT` | References parent instance for sub-orchestrations |

```sql
-- Delta from 0001:
ALTER TABLE instances ADD COLUMN IF NOT EXISTS parent_instance_id TEXT;
```

### Indexes Added

| Index | Table | Column(s) | Purpose |
|-------|-------|-----------|---------|
| `idx_instances_parent` | `instances` | `parent_instance_id` | Efficient child instance lookups |

```sql
CREATE INDEX IF NOT EXISTS idx_instances_parent ON instances(parent_instance_id);
```

### New Procedures

#### `list_children(p_instance_id TEXT)`

Returns all direct child instances of the given parent.

```sql
CREATE OR REPLACE FUNCTION %I.list_children(p_instance_id TEXT)
RETURNS TABLE(child_instance_id TEXT) AS $list_children$
BEGIN
    RETURN QUERY
    SELECT i.instance_id
    FROM %I.instances i
    WHERE i.parent_instance_id = p_instance_id
    ORDER BY i.created_at;
END;
$list_children$ LANGUAGE plpgsql;
```

**Usage**: Called by `ProviderAdmin::list_children()` for hierarchy traversal.

#### `get_parent_id(p_instance_id TEXT)`

Returns the parent instance ID, or NULL if root.

```sql
CREATE OR REPLACE FUNCTION %I.get_parent_id(p_instance_id TEXT)
RETURNS TEXT AS $get_parent$
DECLARE
    v_parent_id TEXT;
BEGIN
    SELECT i.parent_instance_id
    INTO v_parent_id
    FROM %I.instances i
    WHERE i.instance_id = p_instance_id;
    
    IF NOT FOUND THEN
        RAISE EXCEPTION 'Instance not found: %', p_instance_id;
    END IF;
    
    RETURN v_parent_id;
END;
$get_parent$ LANGUAGE plpgsql;
```

**Raises**: `Instance not found: <id>` if instance doesn't exist.

**Usage**: Called by `ProviderAdmin::get_parent_id()` for hierarchy traversal.

#### `delete_instances_atomic(p_instance_ids TEXT[], p_force BOOLEAN)`

Atomically deletes multiple instances and all related data.

```sql
CREATE OR REPLACE FUNCTION %I.delete_instances_atomic(
    p_instance_ids TEXT[],
    p_force BOOLEAN
)
RETURNS TABLE(
    instances_deleted BIGINT,
    executions_deleted BIGINT,
    events_deleted BIGINT,
    queue_messages_deleted BIGINT
) AS $delete_atomic$
DECLARE
    v_instance_id TEXT;
    v_orphan_id TEXT;
    v_instances_deleted BIGINT := 0;
    v_executions_deleted BIGINT := 0;
    v_events_deleted BIGINT := 0;
    v_queue_deleted BIGINT := 0;
    v_count BIGINT;
BEGIN
    -- Handle NULL/empty array
    IF p_instance_ids IS NULL OR array_length(p_instance_ids, 1) IS NULL THEN
        instances_deleted := 0;
        executions_deleted := 0;
        events_deleted := 0;
        queue_messages_deleted := 0;
        RETURN NEXT;
        RETURN;
    END IF;

    -- Safety check: reject if any instance is Running (unless force=true)
    IF NOT p_force THEN
        SELECT i.instance_id INTO v_instance_id
        FROM %I.instances i
        JOIN %I.executions e ON i.instance_id = e.instance_id 
          AND i.current_execution_id = e.execution_id
        WHERE i.instance_id = ANY(p_instance_ids)
          AND e.status = 'Running'
        LIMIT 1;
        
        IF v_instance_id IS NOT NULL THEN
            RAISE EXCEPTION 'Instance % is Running. Use force=true to delete.', v_instance_id;
        END IF;
    END IF;

    -- Lock all instances to be deleted
    PERFORM 1 FROM %I.instances
    WHERE instance_id = ANY(p_instance_ids)
    FOR UPDATE;

    -- Orphan detection: fail if deleting parent would orphan a child
    SELECT i.instance_id INTO v_orphan_id
    FROM %I.instances i
    WHERE i.parent_instance_id = ANY(p_instance_ids)
      AND NOT (i.instance_id = ANY(p_instance_ids))
    LIMIT 1;
    
    IF v_orphan_id IS NOT NULL THEN
        RAISE EXCEPTION 'Orphan detected: instance % has parent in delete list but is not included', v_orphan_id;
    END IF;

    -- Delete in foreign key safe order
    DELETE FROM %I.history WHERE instance_id = ANY(p_instance_ids);
    GET DIAGNOSTICS v_count = ROW_COUNT;
    v_events_deleted := v_count;

    DELETE FROM %I.executions WHERE instance_id = ANY(p_instance_ids);
    GET DIAGNOSTICS v_count = ROW_COUNT;
    v_executions_deleted := v_count;

    DELETE FROM %I.orchestrator_queue WHERE instance_id = ANY(p_instance_ids);
    GET DIAGNOSTICS v_count = ROW_COUNT;
    v_queue_deleted := v_count;

    DELETE FROM %I.worker_queue 
    WHERE work_item::JSONB ? 'ActivityExecute'
      AND (work_item::JSONB->'ActivityExecute'->>'instance') = ANY(p_instance_ids);
    GET DIAGNOSTICS v_count = ROW_COUNT;
    v_queue_deleted := v_queue_deleted + v_count;

    DELETE FROM %I.instance_locks WHERE instance_id = ANY(p_instance_ids);

    DELETE FROM %I.instances WHERE instance_id = ANY(p_instance_ids);
    GET DIAGNOSTICS v_count = ROW_COUNT;
    v_instances_deleted := v_count;

    instances_deleted := v_instances_deleted;
    executions_deleted := v_executions_deleted;
    events_deleted := v_events_deleted;
    queue_messages_deleted := v_queue_deleted;
    RETURN NEXT;
END;
$delete_atomic$ LANGUAGE plpgsql;
```

**Parameters**:
- `p_instance_ids`: Array of instance IDs to delete
- `p_force`: If FALSE, fails if any instance is Running; if TRUE, deletes regardless

**Safety checks**:
1. If `p_force=FALSE`: Rejects if any instance has `status='Running'`
2. Orphan detection: Fails if deleting a parent would orphan a child not in the delete list

**Delete order** (foreign key safe):
1. `history` (events)
2. `executions`
3. `orchestrator_queue`
4. `worker_queue` (ActivityExecute items for these instances)
5. `instance_locks`
6. `instances`

**Usage**: Called by `ProviderAdmin::delete_instances_atomic()`.

#### `prune_executions(p_instance_id TEXT, p_keep_last INTEGER, p_completed_before_ms BIGINT)`

Removes old executions while preserving recent history.

```sql
CREATE OR REPLACE FUNCTION %I.prune_executions(
    p_instance_id TEXT,
    p_keep_last INTEGER DEFAULT NULL,
    p_completed_before_ms BIGINT DEFAULT NULL
)
RETURNS TABLE(
    instances_processed BIGINT,
    executions_deleted BIGINT,
    events_deleted BIGINT
) AS $prune_exec$
DECLARE
    v_current_execution_id BIGINT;
    v_executions_deleted BIGINT := 0;
    v_events_deleted BIGINT := 0;
    v_count BIGINT;
    v_exec_ids_to_delete BIGINT[];
BEGIN
    -- Get current execution (never deleted)
    SELECT i.current_execution_id INTO v_current_execution_id
    FROM %I.instances i
    WHERE i.instance_id = p_instance_id;
    
    IF NOT FOUND THEN
        RAISE EXCEPTION 'Instance % not found', p_instance_id;
    END IF;

    -- Find executions to delete based on filters
    SELECT array_agg(e.execution_id) INTO v_exec_ids_to_delete
    FROM %I.executions e
    WHERE e.instance_id = p_instance_id
      AND e.execution_id != v_current_execution_id  -- Never delete current
      AND e.status != 'Running'                      -- Never delete Running
      AND (p_completed_before_ms IS NULL 
           OR e.completed_at < TO_TIMESTAMP(p_completed_before_ms / 1000.0))
      AND (p_keep_last IS NULL 
           OR e.execution_id NOT IN (
               SELECT e2.execution_id 
               FROM %I.executions e2
               WHERE e2.instance_id = p_instance_id
               ORDER BY e2.execution_id DESC
               LIMIT p_keep_last
           ));

    -- Handle no executions to delete
    IF v_exec_ids_to_delete IS NULL OR array_length(v_exec_ids_to_delete, 1) IS NULL THEN
        instances_processed := 1;
        executions_deleted := 0;
        events_deleted := 0;
        RETURN NEXT;
        RETURN;
    END IF;

    -- Delete history first (child records)
    DELETE FROM %I.history h
    WHERE h.instance_id = p_instance_id
      AND h.execution_id = ANY(v_exec_ids_to_delete);
    GET DIAGNOSTICS v_count = ROW_COUNT;
    v_events_deleted := v_count;

    -- Delete executions
    DELETE FROM %I.executions e
    WHERE e.instance_id = p_instance_id
      AND e.execution_id = ANY(v_exec_ids_to_delete);
    GET DIAGNOSTICS v_count = ROW_COUNT;
    v_executions_deleted := v_count;

    instances_processed := 1;
    executions_deleted := v_executions_deleted;
    events_deleted := v_events_deleted;
    RETURN NEXT;
END;
$prune_exec$ LANGUAGE plpgsql;
```

**Parameters**:
- `p_instance_id`: Instance to prune
- `p_keep_last`: Number of most recent executions to keep (optional)
- `p_completed_before_ms`: Only prune executions completed before this timestamp (optional)

**Safety guarantees**:
- Never deletes the current execution (`current_execution_id`)
- Never deletes Running executions
- Filters are AND'd together if both provided

**Usage**: Called by `ProviderAdmin::prune_executions()`.

## Breaking Changes

None. All changes are additive:
- New column has no NOT NULL constraint
- Existing instances will have `parent_instance_id = NULL` (correct for root instances)
- All existing stored procedures continue to work with existing callers
